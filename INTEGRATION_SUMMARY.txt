HOW TO USE
================================================================================

1. COMPILE OpenSees with MultiSurfCrack2D
   - Follow: https://github.com/RESSLab-Team/OpenSees-Instructions
   - Add your .h and .cpp files to OpenSees source
   - Recompile OpenSeesPy
   - See MULTISURFCRACK2D_INTEGRATION.md for details

2. RUN THE GUI
   python run_gui.bat

3. SELECT MATERIAL
   ② Crack Materials tab → Material type: "MultiSurfCrack2D"

4. CONFIGURE PARAMETERS
   - kn: normal stiffness (210 kN/m default)
   - kt: shear stiffness (5.95 kN/m default)
   - fc, ag, fcl, Acr, etc.: optional (use defaults if unsure)

5. RUN ANALYSIS
   ④ Run tab → Click "▶ Run Analysis"
   - If MultiSurfCrack2D compiled: uses your model
   - If not compiled: falls back to Elastic, still works

TECHNICAL DETAILS
================================================================================

OpenSees Command Generated:
  ops.nDMaterial('MultiSurfCrack2D', mat_id,
                 kn, kt, 0.0,        # De (loading stiffness)
                 kn, kt, 0.0,        # DeU (unloading stiffness)
                 fc, ag, fcl, Acr,   # concrete properties
                 rho_lok, chi_lok,   # interlock parameters
                 rho_act, mu, chi,   # friction/cohesion
                 zeta, kappa, theta, # plasticity parameters
                 w0, cPath)          # crack geometry

Zero-Length Element:
  ops.element('zeroLength', elt_id, nb, na,
              '-mat', mat_id, '-dir', 1, 2)

Node pairs (nb, na):
  - nb = node BELOW crack
  - na = node ABOVE crack
  - Element connects across crack interface
